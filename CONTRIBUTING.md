# Contributing to Resonance

Thanks for helping improve Resonance. Please search existing issues before opening a new one and keep reports focused on a single problem or suggestion.

## Bug reports

Include the Minecraft and loader versions, a clear description of what happened, steps to reproduce it, and the latest log or crash report when relevant. Screenshots or short videos are especially useful for visual and world-generation problems.

## Code changes

1. Create a branch from the appropriate supported-version branch.
2. Keep changes focused and preserve existing Minecraft conventions where possible.
3. Run `./gradlew build` before submitting a pull request.
4. Explain the player-facing behavior and how it was tested.

Do not include generated build output, game saves, logs, or redistributed third-party assets.
